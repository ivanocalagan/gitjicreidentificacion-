1. OBJETIVO DEL PROYECTO
═════════════════════════════════════════════════════════════════════════════

Desarrollar un framework web (DAT-SIN) que garantice anonimización de datos
personales mediante privacidad diferencial, cumpliendo con la Ley 81 de 
Protección de Datos Personales de Panamá (2019).

Problema: Los DBA clonan BD de producción sin enmascaramiento, exponiendo
datos reales (cedulas, nombres, salarios, etc.).

Solución: Sistema automatizado que convierte datos reales en datos sintéticos
estadísticamente realistas, conservando patrones pero eliminando identidades.

3. PRIVACIDAD DIFERENCIAL (DP)
═════════════════════════════════════════════════════════════════════════════

¿QUÉ ES?
Técnica criptográfica que añade ruido controlado a datos sensibles, 
garantizando que no se pueda inferir si un registro individual está en el dataset.

FÓRMULA MATEMÁTICA:
valor_sintético = valor_original_muestra + ruido_laplace(0, sensitivity/ε)

Donde:
- sensitivity = (max - min) / n_rows
- ε (epsilon) = parámetro de privacidad (menor ε = mayor privacidad)

EJEMPLO PRÁCTICO:
Salario original: $2,500
Rango de datos: $500 - $5,000
N filas: 50
sensitivity = (5000-500) / 50 = 90
ε = 1.0 (recomendado Ley 81)
noise_scale = 90 / 1.0 = 90
ruido_laplace = 23.4 (aleatorio)
Salario sintético: 2500 + 23.4 = $2,523.40 ✓

RESULTADO:
- Valor perturbado = diferente del original pero realista
- Estadísticas preservadas = media, desv.est. similares
- Garantía DP = No puedo saber si $2,500 estaba en original


4. ALGORITMO DE EVALUACIÓN DE RIESGO
═════════════════════════════════════════════════════════════════════════════

CLASIFICACIÓN DE COLUMNAS (Heurística por nombre):

Identificadores Directos (+35 puntos c/u):
  nombre, cedula, correo, telefono, direccion, 
  DNI, RUC, NSS, cuenta_bancaria, tarjeta

Cuasi-Identificadores (+15 puntos c/u):
  edad, genero, fecha_nacimiento, carrera, 
  facultad, codigo_postal, municipio, provincia, 
  salario, ingreso, fecha_ingreso, nacionalidad

Atributos Sensibles (+10 puntos c/u):
  saldo_ahorros, monto_prestamo, deuda, balance,
  calificacion_credito, historial, estado_cuenta

Columnas Seguras (0 puntos):
  Resto (id_socio, tipo_cuenta, estado, etc.)

SCORING:
- Cálculo: Suma puntos por cada columna encontrada
- Rango: 0-100 (clampead)
- Clasificación:
  • 70-100: RIESGO ALTO (🔴 rojo #EF4444)
  • 40-69:  RIESGO MEDIO (🟡 amarillo #F59E0B)
  • 0-39:   RIESGO BAJO  (🟢 verde #10B981)

EJEMPLO:
Datos: id_socio, nombre, cedula, correo, carrera, saldo_ahorros
Scoring:
  nombre → +35
  cedula → +35
  correo → +35
  carrera → +15
  saldo_ahorros → +10
  ─────────────────
  Total: 130 → Clampea a 100 → RIESGO ALTO ✓

7. CUMPLIMIENTO LEY 81 PANAMÁ
═════════════════════════════════════════════════════════════════════════════

ARTÍCULOS APLICABLES:

Art. 2: Derecho a la privacidad
  → DAT-SIN garantiza anonimización de datos personales

Art. 5: Principio de finalidad limitada
  → Datos sintéticos solo para desarrollo/pruebas, no producción

Art. 8: Principio de seguridad
  → Privacidad diferencial = garantía matemática
  → Datos nunca persisten en disco durante sesión

Art. 15: Tratamiento de datos sensibles
  → Identificadores directos eliminados (cedula, nombre, email)
  → Reemplazados por valores Faker 100% falsos

DECLARACIÓN EN REPORTE PDF:
"Los datos exportados cumplen con privacidad diferencial epsilon≤1.0 
según Ley 81 de Protección de Datos Personales de Panamá (2019). 
Ningún dato personal real fue almacenado en disco."
